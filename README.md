
Парсер вакансий hh.ru, который сохраняет результаты в CSV-файл с подробной информацией о каждой вакансии.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Использование

```bash
python hh_parser.py <ключевые_слова> [опции]
```

Ключевые слова передаются через пробел и объединяются в один поисковый запрос. Основные опции:

- `-o`, `--output` — путь к CSV-файлу (по умолчанию `vacancies.csv`).
- `-a`, `--area` — код региона (см. API hh.ru `/areas`).
- `--specialization` — код специализации (см. API hh.ru `/specializations`).
- `--industry` — идентификатор индустрии (см. API hh.ru `/industries`).
- `--employment` — тип занятости (full, part, project, volunteer, probation).
- `--schedule` — тип графика (fullDay, shift, flexible, remote, flyInFlyOut).
- `--experience` — требуемый опыт (noExperience, between1And3, between3And6, moreThan6).
- `--page-count` — количество страниц результатов, которые нужно собрать (по умолчанию 1).
- `--per-page` — количество вакансий на странице (до 100).
- `--throttle` — задержка между запросами деталей вакансий, чтобы не перегружать API (в секундах).
  Клиент выставляет тайм-ауты и повторные попытки на сетевые запросы, чтобы надёжнее собирать вакансии.

### Пример

Собрать две страницы вакансий Python для Москвы (код региона 1) и сохранить в `python_jobs.csv`:

```bash
python hh_parser.py python разработчик -a 1 --page-count 2 -o python_jobs.csv
```

## Выходные данные

CSV-файл содержит столбцы:

- Название вакансии
- Название компании
- ИНН компании
- Указанная ЗП (или «—»)
- Ключевые требования (навыки)
- Ссылка на вакансию
- Контактное лицо (если доступно)
- Контакты (если доступны)
- Описание (полностью, без HTML)

Скрипт обращается к публичному API hh.ru: сначала получает список вакансий по заданным фильтрам, затем для каждой вакансии загружает полную карточку, достаёт навыки, описание, контакты и формирует CSV. Ошибки сети или некорректный ответ для конкретной страницы или карточки выводятся в консоль, чтобы парсер мог продолжать обработку остальных вакансий без остановки.
